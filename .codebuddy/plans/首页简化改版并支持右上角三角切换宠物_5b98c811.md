## User Requirements

- 将首页改造成与提供图片一致的简洁样式：顶部标题“首页”、宠物信息行、统计概览、快捷操作、底部导航保持现状。
- 右上角三角用于切换不同宠物，不再使用当前圆点切换方式。
- 页面视觉应更轻量，减少当前冗余区块，信息层级清晰。

## Product Overview

- 首页以“单屏核心信息”为主：先展示当前宠物与切换入口，再展示四项统计概览，最后展示四个快捷操作入口。
- 视觉效果为浅色背景、白色卡片、低干扰分隔，强调可读性与点击区域清晰度。
- 宠物切换交互需直接、稳定，切换后首页内容同步更新。

## Core Features

- 宠物头部区：展示宠物头像/名称、宠物数量标签、右上角三角触发的宠物切换菜单。
- 统计概览区：四宫格展示健康评分、总记录数、疫苗接种、驱虫记录（无数据时显示0）。
- 快捷操作区：保留并收敛为体重记录、疫苗记录、驱虫记录、智能分析四个入口。
- 简化首页结构：移除趋势卡与最近活动等非目标模块，保持页面与示意图一致。

## Tech Stack Selection

- 前端：React + TypeScript（现有）
- 路由：React Router（现有 `/app/pets`）
- 图标：lucide-react（现有）
- 样式：全局 `src/index.css` 中 `pet-home-*` 体系（现有）

## Implementation Approach

- 以 `src/pages/Pets.tsx` 为唯一首页结构入口重构，复用 `useShell()` 提供的 `pets / selectedPetId / setPetId`。
- 将“宠物切换”从 dots 改为右上角三角触发的下拉列表，点击项后调用 `setPetId` 并关闭菜单。
- 将页面区块收敛为“宠物头部 + 统计概览 + 快捷操作”，删除与目标图不一致的复杂区块。
- 统计优先使用安全默认值（0），避免额外请求引入首屏延迟；后续可在同结构内渐进接入真实统计。

## Implementation Notes

- 仅改动首页相关代码，保持 `AppShell` 底部导航和路由不变，控制影响范围。
- 宠物切换菜单需处理空列表、单宠物、点击外部关闭、当前项高亮，避免误触与状态残留。
- 复用现有 `pet-home-*` 命名，避免新增分散样式体系；移动端断点继续沿用 `@media (max-width: 440px)`。
- 保持快捷入口既有跳转路径（`/app/add-record`、`/app/records`）不变，确保行为兼容。

## Architecture Design

- 页面组件层：`Pets.tsx` 负责结构渲染、宠物切换状态、区块数据绑定。
- 样式层：`index.css` 负责新版布局、卡片、下拉菜单、响应式规则。
- 数据层：通过 `useShell` 读取当前宠物上下文；统计先本地占位，后续可平滑扩展 API 数据。

## 设计方案

- 采用简洁轻量风格：白底+浅绿统计容器+低对比边框，减少装饰性阴影。
- 页面从上到下：标题、宠物行、统计概览、快捷操作，保持稳定节奏与足够留白。
- 交互重点为右上角三角切换宠物，菜单层级清晰、点击反馈明确、当前宠物状态可感知。
- 快捷操作卡片采用统一高度与图标色块，形成稳定视觉网格。
